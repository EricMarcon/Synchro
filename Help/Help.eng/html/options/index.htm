Options

Options can be used in the command line or checked on the synchronization form in interactive mode.

Copy options

  • /s: Copy subfolders.
  • /e: Copie empty subfolders. Meaningless without /s.
  • /r: Overwrite read-only files.
  • /d: Preserve newer files. Only older files will be overwritten. If size is different, files will be copied anyway.
  • /Samba: with /d, considers file dates are identical if they differ by less than a second. Allows using different file systems for source and targets, especially Samba NASes.
  • /u: Only copy existing files. No new file will be created.
  • /a : Copy ACLs, including access rights (DACLs), audit parameters and ownership. Rights on files are changed only if the file is changed before.
  • /k : Copy file and folder attributes (read only,...). File attributes are changed only if the file is changed before.
  • /x="List": excludes files and folders according to the patterns.
    • Wildcards * and ? can be used. Separate the patterns by ;. Examples:
      • /x="*\Local Settings\*" : excludes the Local settings folder.
      • /x="*\temp\*;*.bak;*\~*" : excludes the Temp folder, files whose extension is .bak, and files and folders starting with ~.
    • For advanced users, other characters are available:
      • # is for a digit (more restrictive than ?)
      • [A-E] is for all characters between A and E, whatever the case, eventually including accented characters depending on the system locale (accented e are excluded here because they come after E in the sort order)
      • [!B-D] is for all characters except those between B and D.
      • Example: The file C:\aM5b.exe will not be copied with option /x="C:\a[L-P]#[!c-e]*"

Behavior options

  • /n: Do not prompt before creating new files.
  • /y: Do not prompt before deleting files
  • /g: Do not prompt stop processing when errors occur
  • /w: Prompt before closing the window at the end of the synchronization (no effect in console mode)
  • /q: Do not evaluate the folder size before copying it. Faster at the cost of the progress bar (no effect in console mode)
  • /l ="File Name": Log activity in the chosen file

 

Command line only

  • /go launches the synchronization in console mode. Messages are displayed in the active command console. If /n and /y options are used, no user intervention is needed. /g and /q are forced, /w is ignored.

 

Example:

  • Synchro "C:\Tools\My Docs.syn" /go runs the synchronization saved in "My Docs.syn". It starts automatically.
  • Synchro /Source=C:\Source Target=C:\Target /s /e /n /z /l=C:\Synchro.log synchronizes C:\Target on C:\Source. It copies subfolders, even empty, does not prompt before creating files (but prompts before deleting), and deletes extra files in the target. Everything is recorder in the C:\Synchro.log file.
